Summary — what Concord does

Six capabilities, one service.

1

Ingests the workbook

Loads the NIST Cross Mappings Rev. 1.1 workbook (17 sheets) into Postgres. SP.800-53Ar5_assessment is parsed into typed ccf.controls and normalized ccf.framework_mappings; every other tab lands in generic ccf.worksheets / worksheet_rows.

2

Classifies 550+ mapping columns

Resolves each spreadsheet column into one of 26 canonical frameworks — FedRAMP, CMMC, NIST 800-171/172, HIPAA, HITRUST, ISO 27001, SOC 2, CIS v8, NIST CSF, GDPR, StateRAMP, CJIS, MARS-E, AWS / Azure / GCP, CDM, CUI overlay, and more.

3

Models a compliance-ops layer

Beyond the catalog: organizations, systems (FIPS-199 + FedRAMP baseline + ATO status), per-system control implementations, evidence, assessments, POA&Ms, risks, and an append-only audit log.

4

Serves a UI

HTMX + Tailwind server-rendered pages: dashboard, faceted control browser, per-control detail with mappings grouped by framework, framework catalog, generic worksheet viewer, and Postgres full-text search.

5

Publishes a REST API

Everything the UI renders is available as JSON under /api, with Swagger UI at /docs and the schema at /openapi.json.

6

Provides a Typer CLI

ccf ingest · ccf stats · ccf show <identifier> · ccf search <query> · ccf serve

  • cli.py — Typer entrypoint

  • Alembic baseline creating ccf, ccf_raw, ccf_audit schemas

  • 13 enums, 17 tables, GIN indexes on JSONB + tsvector + trigram mappings
